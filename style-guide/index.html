Getting Started

This style guide will help you manage the open-ended creative email development process while maintaining the look and feel of the DICK’S Sporting Goods brand and annual campaigns. But don’t look at it as a set of hard-and-fast rules that handcuff creativity because every campaign or project has its own set of requirements with their own unique solutions. Instead, look at this style guide as a roadmap that helps get you where you’re going. While there is one destination (an effective email that adheres to our brand standards), there are a lot of routes to get there.

When starting a project, it’s best not to reinvent the wheel. It’s most efficient to establish a templated approach that works best for you, meaning you should adapt formats and builds that have worked before and apply new solutions and custom options as necessary.

It is important to remember that "DSG is a branded house, and not a house of brands". Our look and feel as well as our voice should feel like our own and not one that is merely and echo or repitition of our vendors.

Output

Once your design has undergone the rigors of editing and revisions, it’s time to save out your design into a format that email clients and web browsers will understand. To do this, the email needs to be “cut” into smaller chunks. These chunks or slices are necessary for two reasons:

  • Email clients often flag emails containing one big image as spam. To avoid the spam filters, an email should be comprised of multiple images.
  • Cutting the email into smaller images will allow the email to load much faster than one image with a hefty file size. This provides the customer with a much better experience.

Slice Tool

Using the slice tool we are blocking out the areas of the design that will have sperate links & seperate image requirments. Doing so will allow you to set each regions image type as well as make them seperate links.


Save For Web

Use the Save For Web option from the File menu Shift+Command+Option+S after you have used the Slice Tool to cut your design. Within this dialog, you will see your design and the web slices you created.

PS Save For Web Dialog


At this point you need to compress each of your web slices so the final file size of your slices are nice and small. For this, adjust the compression settings that are found along the right side of the Save For Web dialog. Choosing how to output and compress each of the slices is deteminded by what the image is showing.

File Types

  • .gif (Graphic Interchage Format) = animations / flat color.
  • .png (Portable Netowrk Graphic) = flat color and transparency.
  • .jpg / .jpeg (Joint Photographic Experts Group) = full color, small file size.
.GIF

.GIFs are used predominantly for animation, however they can be used for sections of completly flat color, and also support transparency. They can be one of the smallest formats to export, but may also be one of the largest depending on animation and color-depth. Making sure you have the proper color and file size to achieve the design in the output settings is crucial.


.PNG

.PNGs are similar to .GIFs in that they have relativly small file sizes, are used for flat-color, support transparency, and are losseless; however, their transparency is preferable as they can have more color-depth and not suffer from the pixel interpilation, artifacts, and pixel grids of .GIFs.


.JPG

.JPGs offer the widest usage for designs and photos alike. Their small file size is susceptiple to "lossy" compression and every save reduces the qualtiy by adding artifacts.

The photographic uses .JPGs have for photography can't be understated. 80% of designs have flat-color and full-color elements so .JPGs are usually the most well-rounded option to be exported. Keeping in mind that lossy compression can give some undesirable results so it is key to adjust the output settings. When possible, go with a .JPG.

Scalable Design

Scalable design can be defined as any design that works well across both desktop and mobile without using code to adjust table or image sizes, or display or hide content between the two platforms. While we’re using the term “scalable” to describe this approach, you may have also heard these concepts referred to as mobile aware, mobile friendly, device agnostic or mobile first.

Scalable designs are typically the easiest to implement. Since scalable emails don’t adjust the widths of tables or images between devices, and don’t use CSS media queries (we’ll get to those later) to swap content or change the size of text, it’s important to use a number of techniques to keep the content of scalable emails enjoyable not only on desktop, but when viewed on mobile devices.

  • A simple (usually single-column) layout that scales down nicely.
  • Large text that is readable on screens of all sizes.
  • Large, touch-friendly calls-to-action.

Fluid Design

In between scalable and responsive is what we term “fluid” design. Fluid emails use percentage-based sizing to make the width of tables and images adapt to the screen size on which they are viewed, similar to what is known as “liquid” layouts on the web. Fluid emails are similar to scalable ones, in that they don’t purposefully alter the layout or content of an email, but they have the added benefit of having content “flow” to fill space on the screen. For this reason, fluid designs typically work best for text-heavy layouts since there’s less control over how copy and images relate to each other.

  ...Content...
</table>

Responsive Design

Responsive email takes everything from scalable and fluid emails and builds on it by adding more control via CSS media queries. While media queries are also used to help contain fluid layouts, it’s a simple implementation of media queries—nothing too advanced was happening. Responsive email, on the other hand, uses media queries to change the layout of emails, adjust the size of text, images, and buttons, and, in some cases, hide or even swap content between desktop and mobile devices.

@media screen and (max-width: 320px) {
  img[class="hide"]{
    display:none !important;
  }
}

Which is best?

Once you understand the differences between scalable, fluid, and responsive email design, you can determine which strategy works best for your team. If you lack the know-how or resources to implement a responsive campaign, both scalable and fluid strategies are a great way to keep your emails friendly for mobile subscribers. However, if you have the time to go fully responsive, the added control can be exactly the thing needed to set your campaigns apart from the competition.

Scalable Design
Perfect for:
  • Testing the mobile waters.
  • Reliable rendering across clients and devices.
  • Teams with limited resources.
Fluid Design
Perfect for:
  • Simple layouts.
  • Emails with mostly text that can flow.
  • Teams with limited resources.
Responsive Design This is what we use!
Perfect for:
  • Large mobile audiences
  • The most control over layout.
  • Teams with knowledge of media queries.

Code

The design of the email is only as good as the code for users. If the code being used isn't tested or formatted properly it doesn't matter what the design looks like. Having clean code is important for a number of reasons; from being easy to read for the next designer, to any would-be competitor, or even the most finicky of mail clients. Code counts!

Exporting Code

When Photoshop exports your HTML code, it will add a colspan="" attribute to some of your table cell tags <td> if you have any half-width slices in your email. The biggest reason we don’t want to have any colspan="" attributes in the code is because our emails will break in Outlook if we have them in our code. To fix this, we need to place any slices that are not full-width in what’s called a nested table. Nested tables allow us to keep our emails from breaking, plus give us more control over how the email adapts to other screen sizes.

Photoshop's Exported Code

ALT Text

Alt Text is for improved accessability and for when images aren't being shown in the email app or client. This should include all pertinent information about what is being shown both in images and text.

Adding text in the "alt" attribute such as:
alt="PROMO MESSAGE | PRODUCT NAME | SALE $00.00 REG. 00.00 | SHOP NOW"/>

All-caps is used in favor of mixed case for two reasons:

  1. It eliminates the need to worry about strange brand names, product deliniations or model specifics.
  2. It keeps everything at the same weight in the heirarchy.
